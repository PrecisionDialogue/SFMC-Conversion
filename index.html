Documentation and proof of concept for using conversions in SalesForce Marketing Cloud

Welcome the documentation and proof of concept using SFMC Conversions.

Prerequisites

The major prerequisite to implementing conversions is that you have a page hosted on your system that will be the target of all the conversion tracking links in your emails. This page can be hosted on virtually any platform and technology as long as it has the ability to parse URL parameters and store and/or forward those parameter values on to other pages. A proof of concept page that uses only HTML and JavaScript to create conversions is available in this documentation at XXXXXXXX.

Email Configuration

All conversion tracked linked in your email must be created with the conversion="true" attribute as part of the anchor tag. By doing this SFMC will automatically append the required URL parameters to the URL to enable conversion tracking.

Landing Page Configuration

The landing pages must be able to parse the conversion parameters and either store them within a user session scope on either the server or client. The parameter values will need to be used on the actual page where the conversion takes place.

Conversion

On the page where the actual conversion takes place you will need to have access to the user session specific conversion values that were passed to your landing page. You use those parameter values and any transaction specific data that you need to track for the conversion to build an XML string.

After creating the XML string you need to embed the tracking pixel image onto the conversion page (or ensure that it is loaded via scripting or other technology). The tracking pixel URL needs to include the correctly formed XML string as parameter. This will send the conversion data back to SFMC and make it available for reporting.

You can use the fully functional proof of concept example that uses just HTML and JavaScript to fulfil the conversion tracking portion. Create an email in your SFMC account with a bas URL of XXXX and a conversion="true"attribute then visit that link from a sent email, set the appropriate values and click the Perform Conversion Request button to complete the conversion.
